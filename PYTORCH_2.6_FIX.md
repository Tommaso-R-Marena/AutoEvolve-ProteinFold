# PyTorch 2.6 Compatibility Fix

## 🚨 Issue: Quality Gates Test Failures

### Problem

```
FAILED tests/test_architecture_constraints.py::test_model_size
FAILED tests/test_architecture_constraints.py::test_architecture_validity

Error: _pickle.UnpicklingError: Weights only load failed. 
In PyTorch 2.6, we changed the default value of the `weights_only` 
argument in `torch.load` from `False` to `True`.
```

### Root Cause

**PyTorch 2.6** introduced a breaking change where `torch.load()` now defaults to `weights_only=True` for security reasons. This prevents loading checkpoints that contain:
- Custom Python objects
- Dictionaries with metadata
- Config objects
- Performance history
- Generation counters

Our checkpoints save additional metadata beyond just model weights:
```python
checkpoint = {
    'model_state_dict': self.state_dict(),  # ✅ Weights (allowed)
    'config': self.config,                   # ❌ Blocked (dict)
    'generation': self.generation,           # ❌ Blocked (int)
    'performance_history': [...],            # ❌ Blocked (list)
    'metadata': {...}                        # ❌ Blocked (dict)
}
```

### Code Changes

#### Before (Broken)
```python
@classmethod
def load_checkpoint(cls, path: str) -> 'EvolvableProteinFoldingModel':
    """Load model from checkpoint."""
    checkpoint = torch.load(path, map_location='cpu')  # ❌ Fails in PyTorch 2.6
    model = cls(checkpoint['config'])
    model.load_state_dict(checkpoint['model_state_dict'])
    return model
```

#### After (Fixed)
```python
@classmethod
def load_checkpoint(cls, path: str) -> 'EvolvableProteinFoldingModel':
    """Load model from checkpoint.
    
    Note: Uses weights_only=False for compatibility with PyTorch 2.6+.
    This is safe for trusted checkpoint files from our training pipeline.
    """
    # PyTorch 2.6+ requires explicit weights_only=False for checkpoints with metadata
    checkpoint = torch.load(path, map_location='cpu', weights_only=False)  # ✅ Works
    model = cls(checkpoint['config'])
    model.load_state_dict(checkpoint['model_state_dict'])
    model.generation = checkpoint.get('generation', 0)
    model.performance_history = checkpoint.get('performance_history', [])
    return model
```

## 🔒 Security Considerations

### Is `weights_only=False` Safe?

**YES** in our case because:

1. ✅ **Trusted Source**: Checkpoints are generated by our own training pipeline
2. ✅ **Controlled Environment**: GitHub Actions with restricted write access
3. ✅ **No User Input**: Checkpoints are not uploaded by external users
4. ✅ **Repository Protection**: Main branch requires code review
5. ✅ **Automated Generation**: All weights created by verified training scripts

### When is `weights_only=True` Required?

🚨 **CRITICAL for**:
- Loading models from untrusted sources (internet, user uploads)
- Public model hubs without verification
- External datasets or pre-trained weights
- Any checkpoint you didn't create yourself

### Why PyTorch Made This Change

Python's `pickle` (used by `torch.save/load`) can execute arbitrary code during deserialization. Malicious checkpoints could:
- Execute shell commands
- Steal credentials
- Install malware
- Modify system files

PyTorch 2.6+ now requires explicit acknowledgment when loading potentially unsafe data.

## 🔧 Alternative Solutions (Not Used)

### Option 1: Weights-Only Checkpoint Format
```python
# Save only weights
torch.save(model.state_dict(), path)

# Load weights-only
model = Model(config)
model.load_state_dict(torch.load(path, weights_only=True))  # ✅ Safe
```

**❌ Rejected because**: Loses metadata (generation, config, performance history)

### Option 2: Separate Metadata File
```python
# Save weights and metadata separately
torch.save(model.state_dict(), f"{path}.pt")
json.dump(metadata, open(f"{path}.json", 'w'))
```

**❌ Rejected because**: More complex file management, harder to version

### Option 3: SafeTensors Format
```python
from safetensors.torch import save_file, load_file

# Save
save_file(model.state_dict(), path)

# Load
state_dict = load_file(path)  # ✅ Always safe
model.load_state_dict(state_dict)
```

**👍 Recommended for future**: But requires adding `safetensors` dependency
